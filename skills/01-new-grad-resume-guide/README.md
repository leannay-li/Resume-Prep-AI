# New Grad Resume Guide

**For:** Students or recent graduates with zero full-time work experience  
**Use when:** You're building your first resume and unsure about structure, what to include, or positioning  
**Invocation:** `/new-grad-resume-guide`

## What It Does

Provides strategic guidance for someone with no professional full-time experience. Answers foundational questions like:
- What goes where on a resume?
- What counts as "real experience" (internships, projects, coursework)?
- Do I need a Summary? When?
- What are the biggest mistakes new grads make?
- How do I position myself differently for Big Tech vs Startups?

## When to Use It

✅ **Use this if:**
- You're a student or recently graduated
- You have internships, projects, or relevant coursework
- You're unsure about resume structure for someone without job experience
- You want to know what's actually impressive vs what's fluff

❌ **Don't use if:**
- You have 1+ years of full-time work experience (use `experienced-resume-guide` instead)
- You already have a polished resume and just need fine-tuning

## How to Use It

1. Run `/new-grad-resume-guide`
2. Answer the questions:
   - What's your current stage? (Student / Just graduated / Career-switcher)
   - What's your strongest asset? (Internship / Projects / GPA / Coursework)
   - What path are you targeting? (Big tech / Startups / Both)
   - Do you have an existing resume?
3. Get tailored strategic recommendations
4. Next: Move to `summary-writer` or `bullet-writer`

## What You Get

A framework including:
- **Structure guidance** — Recommended order and emphasis for your sections
- **Content strategy** — What's impressive, what's fluff
- **Common mistakes** — What new grads typically get wrong
- **Specific examples** — Good vs bad bullets for your situation
- **Path strategy** — How to position differently for Big Tech vs Startups

## Example Guidance

**You tell it:** "I'm graduating soon with internships and some side projects"

**It responds:**
```
Structure: Contact → Education → Skills → Projects → Work Experience
(Summary is optional—only add if targeting a specific role)

Projects: This is your main section. 2-4 bullets per significant project.
- Internship projects count
- Hackathon projects count
- Side projects count
- Homework assignments DON'T count

Common mistake I see: Listing homework assignments. Replace with real projects.

Big Tech vs Startup:
- Big Tech: Emphasize scale, optimization, testing
- Startup: Emphasize shipping fast, full-stack capability, impact
```

## Key Principles

- **Projects are your proof point** — Not GPA, not coursework
- **Avoid homework assignments** — Use real internships/side projects instead
- **Tech lists should be targeted** — Not "I know 15 languages," but "These 8 are relevant"
- **Be specific with impact** — "Reduced load time from 3s to 800ms" beats "optimized performance"

## Tips

1. **GPA only if 3.7+** — Otherwise omit (looks insecure at lower levels)
2. **Internships > Coursework** — Paid or unpaid internship > relevant course
3. **One project per section** — Don't repeat the same project in "Projects" and "Experience"
4. **Format matters** — Consistent formatting signals attention to detail
5. **Read it out loud** — Does it sound natural? Remove buzzwords
